# Screenshots Needed

Capture these and place them in `docs/screenshots/`. The README references `overview.png`.

| Filename | What to Show |
|---|---|
| `overview.png` | Overview / dashboard page with demo data loaded — KPI cards, expense chart, risk score, monthly trend all visible |
| `transactions.png` | Transactions page sorted by amount descending — category badges and anomaly flags visible |
| `anomalies.png` | Anomalies page showing several flagged transactions with Z-scores |
| `empty-state.png` | The onboarding empty state shown to a new user before their first upload |

## Tips

- Use `frontend/src/assets/demo.csv` to populate the dashboard with realistic data.
- Aim for a browser window width of ~1440px so charts render at full size.
- Use the browser's device toolbar if you want a consistent viewport.
- Crop to just the app window — no browser chrome or desktop in the screenshot.
